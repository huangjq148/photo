# iCloud 风格深色背景设计

## 目标

在保留现有 NOIR 深色品牌与页面结构的前提下，为应用壳内页面加入参考 iCloud 的蓝紫色通透光晕。覆盖首页、登录、注册、相册、收藏、回收站和账户页面；公开分享页继续使用独立纯黑看图背景。新背景应提升空间层次，但不能压过照片、文字和交互控件。

## 视觉方案

- 保留 `#030303` 深黑基底和现有暖白文字体系。
- 使用多个大尺寸、低透明度径向渐变构成环境光：左上偏蓝、右上偏紫、底部偏青。
- 光晕边缘保持充分柔化，避免出现清晰色块或廉价的高饱和霓虹效果。
- 增加极轻微、缓慢的漂移动效，让背景具有 iCloud 式流体感；动效不参与页面布局。
- 在背景上叠加非常克制的顶部柔光，维持 NOIR 的暗部质感。

## 页面与组件影响

### 全局背景

背景由全局 CSS 伪元素绘制，固定在视口内并置于所有内容下方。现有页面无需分别添加装饰节点，也不会改变滚动高度或捕获指针事件。

### 应用外壳

`AppShell` 不再用不透明背景覆盖全局环境光。主导航栏保持现有 `sticky` 定位，只把表面调整为半透明黑色并增强背景模糊，形成深色玻璃层；边框继续沿用现有设计令牌。

### 内容可读性

现有卡片、按钮和文字配色保持不变。光晕透明度受控，正文所在区域仍以深色为主，文本对比度和照片色彩不受明显影响。

## 响应式与可访问性

- 桌面端使用完整的三处光晕和轻微漂移。
- 小屏端降低光晕亮度与位移幅度，避免有限视口内颜色过于集中。
- `prefers-reduced-motion: reduce` 下关闭背景动画。
- 背景层设置 `pointer-events: none`，不影响点击、滚动与键盘操作。
- 不依赖背景颜色表达状态或信息。

## 实现边界

本次仅调整 `app/globals.css` 与必要的 `AppShell` 背景/导航样式，不重做页面、卡片或品牌文案，不加入 Canvas、图片资源或第三方动画库。公开分享路由不在本次视觉范围内。

## 验收标准

1. 应用壳内的首页、认证页和主要功能页均能看到一致的深色蓝紫环境光，公开分享页仍为纯黑背景。
2. 页面仍保持 NOIR 深色调，光晕不会遮挡或抢夺照片主体注意力。
3. sticky 导航具有清晰的深色玻璃效果，滚动时文字保持易读。
4. 背景不产生额外滚动条，不阻挡任何交互。
5. 移动端视觉不过亮；启用“减少动态效果”时背景保持静止。
6. 现有自动化测试与代码检查通过。
