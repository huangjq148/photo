# 公开媒体分享设计

## 目标

修复匿名访客无法加载分享媒体的问题，并为分享者提供有效期、查看和撤销能力。

## 当前问题

- 分享页返回的预览地址指向 `/api/files/...`，该接口要求登录。
- 分享页下载地址指向 `/api/photos/:id/download`，同样要求相册成员身份。
- `PhotoShare` 已有 `expires_at` 和 `revoked_at`，但创建接口和 UI 没有暴露管理能力。
- 每次点击分享都会创建新链接，用户无法查看或撤销历史链接。

## 核心流程

### 创建分享

相册可访问用户在媒体操作菜单中选择有效期：永久、24 小时或 7 天。服务端验证媒体可访问且未删除后创建分享，返回分享 URL 和过期时间。

### 匿名访问

访问 `/share/:token` 时服务端校验分享存在、未撤销、未过期且媒体状态正常。页面中的预览、原图和视频请求统一使用公开分享文件接口，不携带用户 Session。

### 管理分享

媒体详情中显示当前用户创建的分享链接。用户可以复制或撤销链接；撤销不可逆，需要重新分享才能获得新 token。

## 接口设计

### `POST /api/photos/:photoId/shares`

请求：

```json
{ "expiresInHours": 24 }
```

`expiresInHours` 允许 `24`、`168` 或 `null`。响应返回 `id`、`url`、`createdAt`、`expiresAt` 和 `revokedAt`。

### `GET /api/photos/:photoId/shares`

只返回当前用户为该媒体创建的分享记录，按创建时间倒序。

### `DELETE /api/shares/:shareId`

仅创建者可撤销。重复撤销返回成功，保证幂等。

### `GET /api/shares/:token/files/:variant`

`variant` 只允许 `thumbnail`、`preview`、`original`。接口每次请求重新校验分享状态，使用数据库记录定位文件，不接受任意路径。视频 original 支持合法 Range 请求和 206 响应。

## 安全规则

- token 使用密码学安全随机值，URL 中不暴露媒体 ID 或存储路径。
- 文件名参数不参与磁盘路径拼接，避免目录穿越。
- 分享只授权单个媒体，不授权相册或其他尺寸之外的资源。
- 返回 `Cache-Control: private, no-store`，确保撤销后不会被共享缓存长期保留。
- 无效、过期、撤销和媒体删除统一显示“分享已失效”，避免泄露内部状态。

## 页面与交互

- 分享弹窗默认选择 7 天，可切换永久或 24 小时。
- 创建成功后显示可复制 URL、过期时间和撤销按钮。
- 匿名分享页保留当前暗色风格，明确展示文件名、尺寸、所属相册和下载动作。
- 视频分享支持播放、拖动进度和断点 Range 加载。

## 异常场景

| 场景 | HTTP | 用户表现 |
|---|---:|---|
| token 不存在或失效 | 404 | 分享已失效 |
| 媒体已进回收站 | 404 | 分享已失效 |
| 非创建者撤销 | 403 | 无权管理此分享 |
| 非法有效期 | 422 | 请选择有效期限 |
| 非法 Range | 416 | 播放器可重新发起请求 |

## 验收标准

- 无登录 Cookie 的浏览器可预览、下载有效分享中的图片。
- 无登录 Cookie 的浏览器可播放并拖动有效分享中的视频。
- 分享过期、撤销或媒体删除后，页面和全部文件变体立即不可访问。
- 相册成员不能查看或撤销其他用户创建的分享记录。
- 分享管理界面能复制、撤销并重新创建链接。

## 不包含范围

- 相册级批量分享。
- 分享密码和访问次数限制。
- 外部 CDN 和对象存储签名 URL。
