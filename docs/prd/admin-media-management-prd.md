# PRD：Admin 图片与存储文件管理

> 文档状态：需求已确认  
> 版本：V1.0  
> 日期：2026-07-13  
> 适用项目：Photo Management

## 1. 背景

当前系统缺少面向运维人员的集中管理入口，无法直接查看全部业务图片、图片所属用户、业务引用关系，以及项目 `data` 目录中的实际文件。长期运行后，数据库记录与物理文件可能出现不一致，也可能产生未被任何业务数据引用的孤立文件。

本需求新增一个不公开展示入口的 Admin 后台，为管理员提供只读的图片总览，以及受控的存储文件清理能力。

## 2. 目标

### 2.1 业务目标

- 管理员能够集中查看系统中的全部图片及其所属人员。
- 管理员能够了解每张图片的业务引用数量。
- 管理员能够递归查看项目 `data` 目录中的全部文件。
- 管理员能够识别并永久删除引用次数为 0 的文件。
- 所有后台数据和操作只能由通过管理员密码验证的人员访问。

### 2.2 成功标准

- 未登录用户无法查看任何 Admin 页面数据或调用 Admin 接口。
- 图片列表、文件列表及引用次数均可稳定查询。
- 被引用文件无法通过页面或直接请求删除。
- 删除操作不会越过 `data` 目录边界。

## 3. 用户角色

| 角色 | 说明 | 权限 |
|---|---|---|
| 管理员 | 持有后台共享密码的内部运维人员 | 登录后台、查看图片、查看文件、删除未引用文件、退出登录 |
| 普通用户/访客 | 现有业务用户或未登录访问者 | 无权访问 Admin 页面和接口 |

本期只有一种管理员角色，不建立管理员账号、角色或细粒度权限体系。

## 4. 范围边界

### 4.1 本次包含

- `/admin` 入口及鉴权跳转。
- 单一环境变量密码登录。
- 24 小时管理员会话及主动退出。
- 经典后台管理布局。
- 图片列表、搜索、排序、分页及大图预览。
- `data` 目录全部文件的递归列表、搜索、排序及分页。
- 业务图片引用次数与物理文件引用次数展示。
- 引用次数为 0 的文件永久删除。
- 删除前服务端复核与二次确认。
- 加载中、空数据、无权限及操作失败状态。

### 4.2 本次不包含

- 管理员账号体系、多管理员或角色权限。
- 找回、修改管理员密码。
- 图片新增、编辑、删除或恢复。
- 文件上传、下载、重命名、移动或内容编辑。
- 批量删除、回收站或删除恢复。
- 数据导入、导出。
- 操作审计日志。
- 云存储、多存储根目录管理。
- 在公开导航、页面或站点地图中添加 Admin 入口。

### 4.3 后续可迭代

- 多管理员账号及双因素认证。
- 删除审计日志与操作人追踪。
- 孤立文件批量清理及定时扫描。
- 数据库记录与文件系统一致性报告。
- 文件回收站、保留期和恢复机制。
- 文件类型、目录、大小、更新时间等高级筛选。

## 5. 信息架构与路由

| 路由 | 页面 | 访问规则 | 核心内容 |
|---|---|---|---|
| `/admin` | Admin 入口 | 未登录跳转 `/admin/login`；已登录跳转 `/admin/photos` | 不直接渲染业务内容 |
| `/admin/login` | 管理员登录 | 未登录可访问；已登录跳转 `/admin/photos` | 密码输入框、登录按钮、错误提示 |
| `/admin/photos` | 图片管理 | 仅管理员会话可访问 | 图片表格、搜索、排序、分页、预览 |
| `/admin/files` | 文件管理 | 仅管理员会话可访问 | 文件表格、搜索、排序、分页、删除 |

说明：后台不存在公开链接入口，但“隐藏入口”不作为安全措施；安全性必须由服务端会话校验保证。

## 6. 页面与交互设计

### 6.1 登录页

#### 页面结构

- 页面居中显示登录卡片。
- 卡片标题：“管理员登录”。
- 一个密码输入框，默认遮掩输入内容。
- 一个主按钮：“登录”。
- 输入框下方显示校验或登录失败信息。

#### 核心交互

1. 页面打开后，密码输入框自动获得焦点。
2. 管理员输入密码，可点击“登录”或按 Enter 提交。
3. 提交期间按钮进入加载状态，不允许重复提交。
4. 密码正确时创建会话并跳转 `/admin/photos`。
5. 密码错误时清空密码并提示“密码错误”。
6. 已登录管理员访问登录页时直接跳转 `/admin/photos`。

#### 状态与文案

| 状态 | 页面处理 | 文案 |
|---|---|---|
| 密码为空 | 不提交 | 请输入管理员密码 |
| 密码错误 | 停留当前页并清空输入 | 密码错误 |
| 请求失败 | 保留当前页，可重试 | 登录失败，请稍后重试 |
| 提交中 | 禁用输入框与按钮 | 正在登录… |

### 6.2 后台通用框架

采用经典左侧菜单、右侧内容布局。

```text
┌──────────────────────────────────────────────────────────┐
│ 左侧菜单             │ 页面标题                          │
│                      │ 搜索 / 排序                       │
│ • 图片管理           │ ┌──────────────────────────────┐ │
│ • 文件管理           │ │ 数据表格                     │ │
│                      │ │                              │ │
│                      │ └──────────────────────────────┘ │
│ • 退出登录           │ 分页                             │
└──────────────────────────────────────────────────────────┘
```

- 当前菜单项需有明确选中态。
- 退出登录固定置于菜单底部。
- 桌面端为固定侧栏；窄屏可折叠，但本期以桌面管理场景为主。
- 页面首次加载显示骨架或加载状态，不能显示空白区域。

### 6.3 图片管理页

#### 功能描述

展示数据库中 `media_type = image` 的全部 `Media` 记录，包括正常与已进入业务回收站的记录。记录状态不影响其引用次数计算。

#### 查询区

- 一个搜索框，支持按“照片名”或“所属人员邮箱”包含匹配。
- 排序项：上传时间、照片名、引用次数。
- 排序方向：升序、降序。
- 默认排序：上传时间降序。

#### 表格字段

| 字段 | 展示规则 |
|---|---|
| 照片名 | 优先显示 `display_name`，为空时显示 `original_name` |
| 照片 | 展示缩略图；保持比例，加载失败显示占位图 |
| 所属人员邮箱 | 展示 `Media.uploader.email` |
| 被引用次数 | 展示业务引用总数，允许按该字段排序 |

#### 图片预览

- 点击缩略图打开模态预览层。
- 优先加载预览图；必要时可使用原图资源。
- 支持关闭按钮、点击遮罩关闭及 Esc 关闭。
- 预览失败时显示“图片加载失败”，不可导致页面报错。

#### 分页

- 默认每页 20 条。
- 可选择每页 20、50、100 条。
- 显示总条数、当前页和总页数。
- 搜索条件变化后回到第 1 页。

### 6.4 文件管理页

#### 功能描述

以项目根目录下的 `data` 目录为扫描根目录，递归列出其所有普通文件，不限制扩展名或 MIME 类型。目录本身不作为列表记录展示。

现有 `STORAGE_ROOT` 默认为 `./data/storage`，因此其中的原图、预览图和缩略图也属于文件扫描范围；即使未来 `STORAGE_ROOT` 发生变化，本需求仍以项目 `data` 目录为展示边界，超出该边界的文件不纳入本期管理。

#### 查询区

- 搜索框支持按文件名或相对路径包含匹配。
- 排序项：文件名、文件大小、引用次数、最后修改时间。
- 排序方向：升序、降序。
- 默认排序：相对路径升序。

#### 表格字段

| 字段 | 展示规则 |
|---|---|
| 文件名 | 文件基础名称，包含扩展名 |
| 相对路径 | 相对于 `data` 根目录的规范化路径，不显示服务器绝对路径 |
| 文件类型 | 优先根据 MIME/扩展名识别；无法识别时显示“未知” |
| 文件大小 | 自动格式化为 B、KB、MB、GB |
| 最后修改时间 | 使用文件系统最后修改时间，按当前产品统一时区格式化 |
| 被引用次数 | 数据库中对该物理文件规范路径的直接引用数量 |
| 操作 | 删除按钮；引用次数大于 0 时禁用并说明原因 |

#### 删除交互

1. 引用次数为 0 时，“删除”按钮可用。
2. 点击后弹出二次确认框，展示文件名、相对路径及不可恢复提示。
3. 确认文案：“删除后无法恢复，确定永久删除该文件吗？”
4. 管理员确认后，服务端重新获取目标文件并重新计算引用次数。
5. 引用次数仍为 0 时执行物理删除。
6. 删除成功后关闭弹窗、刷新当前列表，并提示“文件已删除”。
7. 取消时不执行任何操作。

## 7. 核心业务规则

### 7.1 管理员密码与会话

- 新增管理员密码环境变量，建议命名为 `ADMIN_PASSWORD`。
- 本地开发环境未配置时，默认值为 `qwerty123`。
- `qwerty123` 不得作为生产环境安全密码；生产环境必须显式配置高强度密码。
- 密码只能在服务端读取和比对，不得写入前端代码、页面数据或接口响应。
- 登录成功后创建 HttpOnly 会话 Cookie，有效期 24 小时。
- Cookie 在生产环境启用 `Secure`，并设置合理的 `SameSite` 属性。
- 退出登录后服务端应立即使当前会话失效。
- 所有 `/admin` 数据接口和变更接口必须独立执行管理员鉴权。
- 登录接口需具备基础防暴力破解措施：按客户端标识限流，连续失败达到阈值后短时拒绝请求；具体阈值可复用项目现有登录限流策略。

### 7.2 业务图片引用次数

图片管理中的“被引用次数”按数据库内指向该 `Media.id` 的直接引用记录数求和：

| 引用来源 | 计数规则 |
|---|---|
| 相册封面 `Album.cover_photo_id` | 每条匹配的相册记录计 1 次 |
| 相册图片关系 `AlbumPhoto.photo_id` | 每条关系记录计 1 次 |
| 收藏关系 `Favorite.photo_id` | 每条收藏记录计 1 次 |
| 单图分享 `PhotoShare.photo_id` | 每条分享记录计 1 次，包括已撤销或已过期记录 |

以下关系不计入图片引用次数：

- `Media.album_id`：图片自身的归属字段，不视为外部引用。
- `Media.uploader_id`：上传者字段，不视为图片被使用。
- `Media.deleted_by`：删除操作人字段，不视为图片被使用。

同一业务记录若通过不同关系引用同一图片，各关系分别计数；计数结果必须为非负整数。

### 7.3 物理文件引用次数

- 文件引用按“数据库字段中保存的规范化文件路径是否指向该物理文件”计算。
- 当前至少检查 `Media.storage_path`、`original_url`、`preview_url`、`thumbnail_url` 对应的本地文件关系。
- 同一条 `Media` 记录的不同字段如果指向同一物理文件，每个字段各计 1 次。
- 比对前应统一绝对/相对形式、路径分隔符及路径规范化规则。
- URL 字段仅在能够确定映射到本地 `data` 文件时计数；外部 URL 不计为本地文件引用。
- 无任何数据库字段指向的文件，引用次数为 0。
- 文件删除前必须实时重新计算，页面上的历史计数不能作为最终删除依据。

### 7.4 文件系统安全规则

- 接口只接受相对于 `data` 根目录的相对路径。
- 服务端拼接并规范化路径后，必须验证最终路径仍位于 `data` 根目录内。
- 拒绝空路径、绝对路径、包含目录穿越的路径，以及解析后越界的路径。
- 不跟随指向 `data` 目录外部的符号链接；符号链接默认不展示、不删除。
- 删除目标必须是普通文件，不允许通过本功能删除目录。
- 不向前端返回服务器绝对路径。
- 文件列表读取失败不应导致整个应用退出，应返回可识别的错误状态。

## 8. 字段设计

### 8.1 登录表单

| 字段 | 类型 | 必填 | 默认值 | 规则 |
|---|---|---:|---|---|
| password | password string | 是 | 无 | 前端不回显；服务端与管理员密码安全比对 |

### 8.2 图片列表响应字段

| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| id | UUID string | 是 | `Media.id` |
| photoName | string | 是 | `display_name` 为空时回退到 `original_name` |

| previewUrl | string | 是 | 预览资源地址 |

| referenceCount | integer | 是 | 业务引用总数 |
| uploadedAt | datetime | 是 | 上传时间，用于默认排序 |

### 8.3 文件列表响应字段

| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| name | string | 是 | 文件名 |
| relativePath | string | 是 | 相对于 `data` 的路径，也是删除操作的文件标识 |
| fileType | string | 是 | 类型无法识别时为 `unknown` |

| modifiedAt | datetime | 是 | 文件系统修改时间 |
| referenceCount | integer | 是 | 物理文件引用次数 |

| 字段 | 类型 | 必填 | 默认值 | 规则 |
|---|---|---:|---|---|

| sortBy | enum string | 否 | 各页面默认值 | 只接受白名单字段 |
| sortOrder | enum | 否 | desc/asc | 只允许 `asc`、`desc` |

## 9. 接口能力建议

以下为产品层面的能力约定，具体 URL 可由开发在不改变行为的前提下调整。

| 方法 | 建议路径 | 用途 | 权限 |
|---|---|---|---|
| POST | `/api/admin/auth/login` | 校验管理员密码并创建会话 | 公开但限流 |
| POST | `/api/admin/auth/logout` | 注销当前管理员会话 | 管理员 |
| GET | `/api/admin/photos` | 查询图片列表 | 管理员 |
| GET | `/api/admin/files` | 查询文件列表 | 管理员 |
| DELETE | `/api/admin/files` | 按相对路径删除单个未引用文件 | 管理员 |

通用响应要求：

- 查询接口返回数据、总条数、当前页和每页条数。
- 参数非法时返回明确的客户端错误，不执行文件扫描或删除。
- 删除冲突（引用次数已变化）与文件不存在应使用可区分的错误码。
- 错误响应不得包含管理员密码、绝对路径、堆栈或内部数据库信息。

## 11. 异常场景

| 场景 | 系统处理 | 提示文案 |
|---|---|---|
| 管理员密码为空 | 阻止提交 | 请输入管理员密码 |
| 管理员密码错误 | 不创建会话，记录失败用于限流 | 密码错误 |
| 登录请求被限流 | 暂时拒绝登录 | 尝试次数过多，请稍后再试 |
| 会话无效或过期 | 接口拒绝；页面跳转登录页 | 登录已失效，请重新登录 |
| 图片列表为空 | 展示空状态 | 暂无图片 |
| 文件列表为空 | 展示空状态 | data 目录中暂无文件 |
| 图片资源加载失败 | 展示占位图 | 图片加载失败 |
| `data` 目录不存在 | 返回受控错误，不自动暴露路径 | 文件目录不存在，请检查服务配置 |
| `data` 目录无读取权限 | 返回受控错误 | 文件目录读取失败 |
| 删除时文件引用数变为大于 0 | 拒绝删除并刷新列表 | 文件已被引用，无法删除 |
| 文件已不存在 | 不重复删除并刷新列表 | 文件不存在，列表已刷新 |
| 文件无删除权限 | 保留文件 | 删除失败：没有文件操作权限 |
| 路径越界或非法 | 拒绝请求并记录安全事件 | 文件路径无效 |
| 删除过程中发生未知错误 | 保留当前列表，允许重试 | 删除失败，请稍后重试 |

## 12. 非功能需求

### 12.1 性能

- 常规数据量下，图片列表查询目标响应时间不超过 2 秒。
- 文件列表首次扫描目标响应时间不超过 3 秒；目录文件量较大时允许采用缓存、增量索引或服务端分页优化。
- 搜索、排序和分页不得一次性向浏览器返回全部记录。
- 引用次数应避免逐行查询造成 N+1 问题。

### 12.2 安全

- 管理密码不得进入客户端 bundle、日志或错误信息。
- 登录接口必须限流，并使用抗时序差异的密码比较方式。
- 管理 Cookie 必须为 HttpOnly，生产环境必须为 Secure。
- 所有写请求应采用项目统一的 CSRF 防护策略或严格的 SameSite/来源校验。
- 严格防御路径穿越、符号链接越界和 TOCTOU 删除风险。
- 删除前的鉴权、路径校验和引用复核必须在服务端完成。

### 12.3 可用性与兼容性

- 支持项目当前定义的主流桌面浏览器版本。
- 所有按钮具备 hover、focus、disabled 和 loading 状态。
- 键盘可完成登录、菜单切换、预览关闭和删除确认。
- 关键文本及操作控件满足基础对比度和焦点可见性要求。

### 12.4 可维护性

- 业务图片引用和物理文件引用应使用两个独立、命名清晰的计算逻辑。
- `data` 根目录应在服务端集中解析，避免各接口自行拼接。
- 引用来源新增时，应同步更新引用计数规则及测试用例。

## 13. 验收标准

### 13.1 登录与权限

1. 未登录访问 `/admin` 时，跳转至 `/admin/login`。
2. 未登录访问 `/admin/photos` 或 `/admin/files` 时，跳转至登录页且不泄露页面数据。
3. 使用正确环境变量密码登录后，创建管理员会话并进入 `/admin/photos`。
4. 输入错误密码时，不创建会话，清空输入框并显示“密码错误”。
5. 登录状态持续 24 小时后失效，再访问后台需重新登录。
6. 点击退出登录后，当前会话立即失效并返回登录页。
7. 未登录直接请求任一 Admin 查询或删除接口时，服务端拒绝访问。
8. 连续多次提交错误密码达到限流阈值后，登录请求被短时拒绝。

### 13.2 图片管理

9. 图片列表仅展示 `media_type = image` 的记录，不混入视频。
10. 每条记录正确显示照片名、缩略图、上传者邮箱和业务引用次数。
11. `display_name` 为空时，照片名回退显示 `original_name`。
12. 输入照片名或邮箱关键字后，仅展示包含匹配的记录。
13. 按引用次数升序或降序切换后，结果顺序正确。
14. 切换 20、50、100 条分页规格时，每页条数及总数正确。
15. 点击缩略图可打开预览；资源失败时显示错误占位且列表仍可操作。
16. 某图片分别存在 2 条相册关系、1 条收藏和 1 条分享时，其业务引用次数显示为 4；图片自身的 `album_id` 不额外计数。

### 13.3 文件管理

17. 文件列表递归展示项目 `data` 目录及子目录中的全部普通文件，不展示目录本身。
18. 每条文件记录正确显示文件名、相对路径、类型、大小、修改时间和物理文件引用次数。
19. 页面和接口响应中不出现服务器绝对路径。
20. 输入文件名或相对路径关键字后，仅展示包含匹配的文件。
21. 数据库中没有任何路径字段指向的文件，其引用次数显示为 0，删除按钮可用。
22. 引用次数大于 0 的文件删除按钮不可用，并能说明不可删除原因。

### 13.4 删除与安全

23. 点击可删除文件的“删除”后，弹窗展示文件名、相对路径及不可恢复提示。
24. 在确认弹窗点击取消时，文件仍存在且列表不发生删除变化。
25. 确认删除引用次数为 0 的文件后，物理文件被永久删除，列表移除该记录并显示成功提示。
26. 打开确认弹窗后若文件新增了数据库引用，确认删除时服务端拒绝操作并刷新引用次数。
27. 文件已被其他进程删除时，系统提示文件不存在并刷新列表，不返回未处理异常。
28. 使用 `../`、绝对路径或编码后的越界路径发起删除请求时，服务端拒绝且不影响 `data` 外文件。
29. `data` 内的符号链接若指向目录外部，不在列表展示且不能通过删除接口操作其目标。
30. 删除目录路径时，服务端拒绝操作，不递归删除目录内容。

## 14. 发布前检查

- 已使用真实 `data` 目录样本验证原图、预览图、缩略图及非媒体文件的引用计数。
- 已验证大文件数量下的扫描时间及内存占用。
- 已完成路径穿越、符号链接越界、并发新增引用和重复删除测试。
- 已确认数据库备份及文件备份策略，因为文件删除不可恢复。
